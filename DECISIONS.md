# Decisions Log

## 001 — pnpm вместо npm
Выбрал: pnpm workspaces.
Отверг: npm workspaces, yarn.
Почему: запрет фантомных зависимостей; в монорепо это критично.

## 002 — tsconfig.base.json в корне
Выбрал: единый базовый конфиг, пакеты наследуют через extends.
Отверг: копия tsconfig в каждом пакете.
Почему: один источник правды; разъезжающиеся конфиги = скрытые баги.

## 003 — внутренние пакеты без сборки
Выбрал: exports указывает на src/, компилирует потребитель (Vite).
Отверг: build-шаг с dist/ в каждом пакете.
Почему: нет оркестрации watch-режимов; сборка нужна только публикуемым пакетам.

## 004 — react как peerDependency в @portal/ui
Выбрал: peerDependencies.
Отверг: dependencies.
Почему: две копии React ломают хуки; UI живёт на React хозяина.

## 005 — feature-based структура в apps/web
Выбрал: src/app + src/features + src/shared.
Отверг: папки по типам (components/, hooks/, utils/).
Почему: код группируется по смыслу; фичу можно удалить одной папкой;
направление зависимостей контролируемо (features не знают друг о друге).

## 006 — tsc --noEmit перед vite build
Выбрал: проверка типов отдельным шагом в build.
Отверг: полагаться на Vite.
Почему: Vite/esbuild типы не проверяет, а вырезает; без tsc битые типы уедут в прод.

## 007 — Input как композиция поля (label + input + error)
Выбрал: label обязателен в пропсах, ошибка и aria внутри компонента.
Отверг: голый стилизованный <input>, label отдельным компонентом.
Почему: связка htmlFor/id/aria-describedby не должна собираться руками
в каждой фиче; доступность принуждается через API, а не через ревью.

## 008 — стилизация ошибки через [aria-invalid], не через класс
Выбрал: селектор по aria-атрибуту.
Отверг: модификатор .ui-input--error.
Почему: один источник правды — нельзя показать ошибку визуально,
не сообщив её скринридеру.

## 009 — нативный select вместо кастомного дропдауна
Выбрал: <select> + appearance:none + своя стрелка.
Отверг: кастомный listbox.
Почему: клавиатура, aria и мобильный UX бесплатно от платформы;
кастомный оправдан только при поиске/мультивыборе/иконках в опциях.

## 010 — внутренний Field с render prop
Выбрал: общая обвязка label/error/aria в приватном компоненте.
Отверг: копипаста обвязки в Input и Select; cloneElement.
Почему: один источник правды для a11y-связки; render prop явный
и типобезопасный; Field не в public API — деталь реализации.

## 011 — своя модалка на createPortal вместо нативного <dialog>
Выбрал: портал + focus trap руками.
Отверг: <dialog> (showModal).
Почему: учебная цель — механика trap'а; полный контроль стилей.
Честно: в новом проде <dialog> — равнозначный или лучший выбор.

## 012 — паттерн latest ref для onClose
Выбрал: onCloseRef + эффект с deps [open].
Отверг: onClose в deps (перезапуск trap на каждый рендер);
onClose вне deps (stale closure).
Почему: подписка редкая, чтение свежее — ref разводит эти две частоты.
